# 简介

这个博客支持 Markdown 和 MDX 格式的文章。Markdown 是一种轻量级标记语言，让你可以用纯文本格式编写文档。

GitHub Flavored Markdown (GFM) 是我们使用的版本，代码块支持语法高亮。

## 什么是 Markdown？

Markdown 是一种在 Web 上样式化文本的方式。你可以控制文档的显示格式，比如将文字设置为粗体或斜体、添加图片、创建列表等等。

## 语法指南

### 标题

```markdown
# 这是一级标题

## 这是二级标题

### 这是三级标题

#### 这是四级标题

##### 这是五级标题

###### 这是六级标题
```

### 文本样式

```markdown
*这是斜体文本*
_这也是斜体文本_

**这是粗体文本**
__这也是粗体文本__

***这是粗斜体文本***
___这也是粗斜体文本___

~~这是删除线文本~~
```

### 列表

#### 无序列表

```markdown
- 项目 1
- 项目 2
  - 子项目 2.1
  - 子项目 2.2
- 项目 3
```

#### 有序列表

```markdown
1. 第一项
2. 第二项
3. 第三项
```

### 链接

```markdown
[链接文本](https://example.com)
[带标题的链接](https://example.com "鼠标悬停显示的标题")
```

### 图片

```markdown
![替代文本](图片URL)
![带标题的图片](图片URL "图片标题")
```

### 引用

```markdown
> 这是一个引用块
>
> 可以有多行
>
> > 还可以嵌套
```

### 代码块

#### 行内代码

```markdown
使用 `反引号` 包裹代码
```

#### 代码块

````markdown
```javascript
function hello() {
  console.log('Hello, World!');
}
```
````

### 表格

```markdown
| 表头 1 | 表头 2 | 表头 3 |
|--------|--------|--------|
| 单元格 1 | 单元格 2 | 单元格 3 |
| 单元格 4 | 单元格 5 | 单元格 6 |
```

显示效果：

| 表头 1 | 表头 2 | 表头 3 |
|--------|--------|--------|
| 单元格 1 | 单元格 2 | 单元格 3 |
| 单元格 4 | 单元格 5 | 单元格 6 |

### 分隔线

```markdown
---
```

### 任务列表

```markdown
- [x] 已完成的任务
- [ ] 未完成的任务
- [ ] 待办事项
```

显示效果：

- [x] 已完成的任务
- [ ] 未完成的任务
- [ ] 待办事项

## 最佳实践

1. **保持简洁**：Markdown 的优势在于简洁，不要过度使用格式
2. **结构清晰**：合理使用标题和列表，让文章层次分明
3. **代码规范**：代码块指定语言，获得更好的高亮效果
4. **图片优化**：使用适当大小的图片，提高加载速度
